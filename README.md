## ✨ Features

- **📱 Progressive Web App** — install on any device
- **🎥 WebRTC Calls** — peer-to-peer audio/video
- **⚡ Go Signaling Server** — lightweight and blazing-fast signaling via WebSockets
- **🗄️ Type-Safe SQL** — database queries generated by `sqlc` — zero runtime surprises
- **🐳 One-Command Deploy** — full stack up with `docker-compose up`
- **🔒 Secure by Design** — STUN/TURN support for NAT traversal

## 🛠️ Tech Stack

### Backend

| Technology            | Role                                               |
| :-------------------- | :------------------------------------------------- |
| **Go 1.23+**          | Signaling server, routing, business logic          |
| **Gin**               | HTTP router and middleware framework               |
| **gorilla/websocket** | Real-time signaling channel for WebRTC negotiation |
| **sqlc**              | Compile-time type-safe SQL query generation        |
| **pgx/v5**            | High-performance PostgreSQL driver                 |
| **godotenv**          | Environment configuration from `.env` files        |

### Frontend

| Technology       | Role                                              |
| :--------------- | :------------------------------------------------ |
| **HTML5**        | 3 pages — lobby, call view, custom 404            |
| **CSS3**         | Responsive design, mobile-first layout            |
| **Vanilla JS**   | WebRTC API, WebSocket client, UI logic (`app.js`) |
| **Lucide Icons** | Lightweight SVG icon library                      |
| **PWA Manifest** | Installable app with home screen icons            |

### Infrastructure

| Technology           | Role                                           |
| :------------------- | :--------------------------------------------- |
| **PostgreSQL**       | Room data and session storage                  |
| **Docker & Compose** | Containerized backend + database + TURN        |
| **Coturn**           | Self-hosted TURN/STUN server for NAT traversal |

## 📐 Architecture

```
┌──────────────┐         WebSocket          ┌──────────────────┐
│              │◄──────────────────────────►│                  │
│   Browser A  │         Signaling          │   Go Server      │
│   (Caller)   │   (SDP Offer/Answer/ICE)   │   (Gin + WS)     │
│              │                            │   :8080          │
└──────┬───────┘                            └──┬───────────┬───┘
       │                                       │           │
       │ Peer-to-Peer Media (audio/video)      │           │  SQL (sqlc + pgx)
       │                                       │           │
       │                                       │  ┌────────▼─────────┐
       │                                       │  │                  │
       │                                       │  │   PostgreSQL     │
       │            WebSocket                  │  │   :5432          │
       │            Signaling                  │  │                  │
       │      (SDP Offer/Answer/ICE)           │  └──────────────────┘
┌──────▼───────┐                               │
│              │◄──────────────────────────────┘
│   Browser B  │
│   (Callee)   │
│              │
└──────────────┘
```

> The Go server only handles signaling (exchanging SDP and ICE candidates).
> Media flows directly between peers via WebRTC when possible (STUN).
> When direct connection fails (strict NAT/firewall), Coturn relays the media (TURN).

## 🚀 Install and Run

### Prerequisites

- **Go** 1.23+
- **Docker** & **Docker Compose**
- **sqlc** (`go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest`)

### 1. Clone the repository

```bash
git clone https://github.com/zkqw3r/Jitter.git
cd Jitter
```

### 2. Configure Environment

Ensure you replicate all `*.env.example` files into local `.env` files and provide your specific configurations before starting the server.

### 3. Start with Docker Compose

```bash
docker-compose up --build
```

### 🌐 Access Points

| Service      | Address               |
| :----------- | :-------------------- |
| **Main App** | http://localhost:8080 |
| **Database** | `localhost:5432`      |

### 4. (Alternative) Run Locally

```bash
# Start only the database
docker-compose up db -d

# Generate sqlc code
sqlc generate

# Install Go dependencies
go mod tidy

# Run the server
go run ./cmd/server
```

### 🎉 **Open in browser:** [http://localhost:8080](http://localhost:8080)

## 📱 PWA Installation

Jitter is a fully installable Progressive Web App:

1. Open `Open localhost:8080 (or your deployed domain)` in Chrome / Safari / Edge
2. Click the **"Install"** prompt (or use browser menu → "Add to Home Screen")
3. Launch Jitter from your home screen — full-screen, no browser UI
